# 旅游多智能体规划器 - 运行指南

## 项目简介

这是一个基于多智能体协作的智能旅行规划系统，支持流式生成每日行程、展示腾讯地图路线、输出旅行手册。

## 环境要求

- **Python**: 3.12（通过 conda 管理）
- **Node.js**: 18+
- **包管理器**: npm 或 yarn

## 第一步：创建并激活 conda 环境

如果你还没有 `py312` 环境，执行：

```bash
conda create -n py312 python=3.12 -y
```

激活环境：

```bash
# Linux/macOS
source ~/anaconda3/etc/profile.d/conda.sh
conda activate py312

# Windows (在 PowerShell 或 cmd 中)
conda activate py312
```

如果提示需要运行 `conda init`，先执行：

```bash
conda init
# 重新打开终端窗口
conda activate py312
```

## 第二步：安装 Python 依赖

```bash
cd travel_multi_agent_planner-fix-agent-unhandled-attractions
pip install -r requirements.txt
```

或者使用 conda run（无需激活环境）：

```bash
conda run -n py312 pip install -r requirements.txt
```

## 第三步：配置环境变量

复制配置文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的 API Key：

```env
DASHSCOPE_API_KEY=你的百炼 API Key
BAILIAN_MODEL=qwen-plus
TRAVEL_APP_MODE=online
TENCENT_MAP_SERVER_KEY=你的腾讯位置服务服务端 Key
TENCENT_MAP_JS_KEY=你的腾讯地图前端 JS Key
```

> **说明**：
> - `DASHSCOPE_API_KEY`: 阿里云百炼大模型的 API Key
> - `TENCENT_MAP_SERVER_KEY` 和 `TENCENT_MAP_JS_KEY`: 腾讯位置服务的 Key，如果只有一个 Key 可以填相同的值

## 第四步：安装前端依赖

```bash
cd frontend
npm install
```

## 第五步：启动服务

需要**两个终端**分别启动后端和前端。

### 终端 1：启动后端

```bash
# 激活 conda 环境
conda activate py312

# 启动 FastAPI 服务
uvicorn backend.main:app --reload --port 8000
```

后端启动成功后，访问 http://localhost:8000/api/health 应返回 `{"status":"ok"}`。

### 终端 2：启动前端

```bash
cd frontend
npm run dev
```

前端启动成功后，访问 http://localhost:5173

## 第六步：使用系统

1. 打开浏览器访问 **http://localhost:5173**
2. 填写旅行信息：
   - 出发地：如 `上海`
   - 目的地：如 `南京`
   - 天数：如 `3`
   - 预算：如 `1500`
   - 兴趣：可多选 `文化`、`美食`、`自然` 等
3. 点击生成，稍等片刻查看流式生成的行程结果

## 备用启动方式：Streamlit 版本

如果不使用前后端分离模式，也可以用旧版 Streamlit 入口：

```bash
conda activate py312
streamlit run streamlit_app.py
```

浏览器访问 http://localhost:8501

## 常见问题

### 1. `conda activate` 命令不可用

```bash
# 先初始化 conda
conda init

# 重新打开终端后再激活
conda activate py312
```

### 2. 前端 `vite` 找不到

```bash
cd frontend
npm install
npm run dev
```

### 3. API Key 相关错误

检查 `.env` 文件中的 Key 是否正确填写，确保：
- 百炼 API Key 有效且有余额
- 腾讯位置服务 Key 已开通相关服务

### 4. 后端启动端口被占用

```bash
# 指定其他端口
uvicorn backend.main:app --reload --port 8001
```

同时需要修改前端 API 配置，将请求地址改为对应的端口。

## 项目结构

```
travel_multi_agent_planner-fix-agent-unhandled-attractions/
├── backend/                  # FastAPI 后端
│   ├── main.py              # 入口，CORS 配置
│   └── routers/
│       ├── plan.py         # POST /api/plan（SSE 流式规划）
│       ├── cases.py         # GET  /api/cases（历史案例）
│       └── status.py        # GET  /api/status/{case_id}
├── frontend/                 # React 前端
│   └── src/
│       ├── components/
│       │   ├── HomeView.tsx    # 输入表单
│       │   ├── LoadingView.tsx # 流式进度
│       │   ├── ResultView.tsx  # 行程结果
│       │   └── TripMapView.tsx # 地图展示
│       └── api/
│           └── client.ts       # API 客户端
├── travel_multi_agent_planner/  # 核心 Agent 逻辑
│   ├── agents/               # 各类型 Agent
│   ├── providers/           # API 提供者（百炼/腾讯）
│   └── orchestrator.py       # 编排器
├── streamlit_app.py          # Streamlit 旧版入口
└── requirements.txt          # Python 依赖
```

## API 端点

| 方法 | 路径                      | 说明             |
|------|---------------------------|------------------|
| GET  | `/api/health`             | 健康检查         |
| GET  | `/api/config`             | 获取前端配置     |
| POST | `/api/plan`              | 发起规划（SSE）  |
| GET  | `/api/cases`             | 历史案例列表     |
| GET  | `/api/status/{case_id}`  | 查询案例状态     |
